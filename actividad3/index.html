1. Ventajas y desventajas de tipos de apps

Tipo de App Ventajas Desventajas
Nativa
  • Puede acceder a todas las funcionalidades del dispositivo.
  • Funcionan offline.
  • Son superiores gráficamente a las otras por aprovechar su plataforma.
  • Debe ser desarrollada exclusivamente para un OS.
  • Requiere mucho tiempo y dinero para desarrollar.
  • Su código no puede ser usado en otras plataformas.
Web
  • No requiere procesos complejos de actualización.
  • Se puede usar el mismo código fuente para todas las plataformas.
  • Es sencillo y barato de desarrollar.
  • No puede usar las funcionalidades de cada dispositivo.
  • No puede ser usada sin conexión a internet.
  • No se puede promocionar en los app stores.
Híbrida
  • Pueden acceder a casi todas las funcionalidades del dispositivo.
  • Se puede usar el mismo código fuente en todas las plataformas.
  • Se pueden promocionar en los app stores.
  • Algunas características no pueden ser compartidas entre plataformas.
  • Gráficamente son inferiores a las apps nativas.
  • Inconsistencias entre plataformas.

2. Modelos de comercialización

Modelo características
Gratuito
  • Puede ser monetizado mediante anuncios (ads).
  • Al ser gratis, muchos usuarios prefieren descargarla.
  • No se obtiene ganancia directamente de los usuarios.
  • Requiere una gran cantidad de usuarios para generar ganancias sustanciales.
Pago
  • Se paga una vez por descarga.
  • Genera ganancias directamente de los usuarios.
  • Se debe ofrecer un buen producto, o la gente no querrá pagarlo.
  • Se debe promocionar muy bien.
Subscripción
  • Genera de ganancias de forma regular.
  • Permite la promoción por medio de "Free trial".
  • Muy buen modelo para apps que promuevan el uso recurrente.
Freemium
  • La descarga es gratis pero cuenta con contenido que se obtiene mediante pago.
  • Se debe mantener a los usuarios interesados en adquirir el contenido pago.
  • Genera buenas ganancias si se tiene una base de usuarios grande.
